## Pre-Launch Checklist

Before publishing the page publicly:

- [ ] Build is tested and working in browser
- [ ] Cover image uploaded (630×500+)
- [ ] At least 3 screenshots uploaded
- [ ] Full description written and formatted
- [ ] All relevant tags added (prefer suggested tags)
- [ ] Platform set to HTML5 only
- [ ] Embed settings configured and tested
- [ ] Controls clearly documented
- [ ] Language set to English only

## Publishing Strategy

1. **Initial Release**: Set to "Draft" or "Restricted" first
2. **Test**: Share restricted link with friends to test embed
3. **Polish**: Fix any issues found during testing
4. **Launch**: Change to "Public" when everything is ready
5. **Post to**: Release Announcements board after launch
6. **Updates**: Use itch.io devlog feature for changelog posts

## Maintenance

- Update version string in description after each build
- Post major updates to devlogs
- Respond to community feedback
- Keep screenshots current with latest version

**Important**: Follow [itch.io Quality Guidelines](https://itch.io/docs/creators/quality-guidelines) strictly:
- Don't select platforms you don't support
- Use only relevant tags from suggested list
- Provide accurate metadata
- Don't republish the page to "boost" visibility
- Upload files directly to itch.io (don't use external links)
